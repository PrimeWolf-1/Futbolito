# Learner Profile

## Who They Are
Rudy O Castillo Espino. Entrepreneur and student based in Los Angeles. Founder of PrimeWolf Media Group (AI-powered social media automation agency, currently in development). Also pursuing cybersecurity. Learns by building real projects with AI tools. Came to this hackathon to learn spec-driven development properly — specifically because a past trading terminal project (ORION) reached v15 in a messy, unplanned way and he wants a repeatable process to apply to all future work. The hackathon project (Futbolito) is the practice run. The process is the real goal.

## Technical Experience
**Level:** Intermediate beginner — knows enough to guide AI and catch problems, does not write code from scratch by hand.

**Approach:** Builds single-file HTML apps — everything in one `.html` file with embedded CSS and JavaScript. Uses Claude as primary development partner.

**API/library integrations completed:**
- Groq API (AI)
- Twelve Data API (financial data)
- TradingView Lightweight Charts (trading terminal UI)

**AI coding agent experience:** Extensive use of Claude for development. Has not used a structured, spec-driven workflow before — previous approach was long sessions with no planning documents.

**What they want to explore:** Formal planning artifacts (scope doc, PRD, technical spec) and multi-session context management strategies.

## Learning Goals
- Learn how to plan a project properly before touching code
- Understand how to write a scope doc, PRD, and technical spec so every future project starts with a blueprint
- Learn how to manage context across sessions so AI stays productive without restarting from zero
- Walk away with a reusable process applicable to ORION, PrimeWolf agency projects, and future builds

## Creative Sensibility
Not a heavy consumer of games or apps (plays Marvel Rivals occasionally). Enjoys variety in music. Aesthetic leans old school — simple, raw, authentic, handmade-feeling. For Futbolito specifically: wants it to feel like the original street game — built with whatever materials were available (wood, nails, string, marble). The vibe should feel grounded, like you're back in the neighborhood in El Salvador. Not polished or corporate. However, appreciates clear craft and effort when something deserves a premium feel — the goal is authentic, not lazy.

## Prior SDD Experience
Has seen both extremes firsthand:
- **ORION (trading terminal):** No planning, built with a partner as they went — reached v15 and feels disorganized.
- **MIA:** Did upfront planning (brand identity, competitive analysis, tech stack) before building — noticeably smoother.

Has never used formal planning artifacts (PRD, technical spec). This hackathon is the first end-to-end structured planning experience. `/reflect` quiz should treat him as someone who understands *why* planning matters from lived experience, but is new to the formal vocabulary and document types.
